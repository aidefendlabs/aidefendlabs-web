AI Defense Knowledge Base

Comprehensive Coverage Across the AI Threat Landscape

From prompt injection to autonomous agent swarms — a growing knowledge base of defensive techniques with implementation guidance and code examples for every major AI attack surface.

Prompt Injection

The #1 LLM threat. Attackers manipulate model behavior through crafted inputs that override system instructions.

  • AID-H-017 System Prompt Hardening
  • AID-H-027 Closed-Loop Prompt Injection Detector Hardening
  • AID-D-001 Adversarial Input & Prompt Injection Detection

Agentic AI Security

Autonomous agents introduce new risks — unauthorized actions, goal drift, and privilege escalation across tool chains.

  • AID-H-018 Secure Agent Architecture
  • AID-H-019 Tool Authorization & Capability Scoping
  • AID-M-009 Agent Autonomy & Authority Governance

MCP & Tool Security

Model Context Protocol expands the attack surface. Tool poisoning, registry spoofing, and TOCTOU attacks threaten agent workflows.

  • AID-H-025 Tool & MCP Resolution Integrity
  • AID-H-029 MCP & Tool Client Security Hardening
  • AID-D-003.004 Tool-Call Sequence Anomaly Detection

RAG Poisoning Defense

Attackers inject malicious content into vector stores and knowledge bases, corrupting retrieval-augmented generation pipelines.

  • AID-H-021 RAG Index Hygiene & Signing
  • AID-D-014 RAG Content & Relevance Monitoring
  • AID-R-005 Rapid Vector Index Rollback & Quarantine

Data Poisoning & Supply Chain

Compromised training data and untrusted model artifacts undermine AI integrity from the foundation up.

  • AID-H-003.003 Dataset Supply Chain Validation
  • AID-H-002.007 External Media & Document Provenance Verification
  • AID-D-004.001 Static Artifact Hash & Signature Verification

AI-Generated Code Safety

AI coding assistants can produce vulnerable or malicious code. Admission controls prevent unsafe code from reaching production.

  • AID-H-026 Unsafe Code Execution Prevention
  • AID-H-032 AI-Generated Code Admission Control & Safe Promotion
  • AID-I-001.005 Pre-Execution Behavioral Analysis in Sandboxes

Multi-Agent & Swarm Security

When agents collaborate autonomously, rogue actors can infiltrate the swarm. Detect compromised agents before they cascade.

  • AID-D-011.002 Inter-Agent Security & Consensus Monitoring
  • AID-D-011.003 Agent Population Drift & Rogue-Instance Escalation
  • AID-D-016 Rogue Agent Discovery, Reputation & Quarantine

AI Deception & Honeypots

Turn the tables on attackers. Deploy decoy AI services, canary tasks, and honey data to detect and study adversaries in real time.

  • AID-DV-001 Honeypot AI Services & Decoy Models/APIs
  • AID-DV-005 Decoy Agent Behaviors & Canary Tasks
  • AID-DV-002 Honey Data, Decoy Artifacts & Canary Tokens

AI Memory & State Security

Persistent agent memory is a new attack surface. Poisoned memories can alter agent behavior long after the initial compromise.

  • AID-M-002.004 Trust-Tiered Memory/KB Write-Gate
  • AID-I-004.005 Memory TTL, Staleness Decay & Forced Forgetting
  • AID-I-004.003 Cryptographic Memory Integrity
